## 修复续报率丢失 + 优化变动概览展示
- **根因**: `calibrator.py` 中 `load_old_ledger_manual_fields()` 用 `'UV台帳'`(繁体帳) 匹配工作表，而实际 sheet 名为 `'UV台帐'`(简体帐)，导致该函数返回空，所有手动字段(含 T/AC/AL 是否续秋)全部丢失
- **修复**: 新增 `_get_uv_sheet()` 同时兼容 `'UV台帐'` 与 `'UV台帳'`，并统一替换 `load_old_ledger_subject_data`、`load_old_ledger_manual_fields`、`generate_calibrated_ledger` 的 sheet 查找逻辑
- **续秋逻辑增强**: `calibrate()` 中是否续秋优先以秋季订单(`fall_data`)为真相源，无秋季订单时继承原台帐 T/AC/AL 列；新增 `_is_continue_fall()`/`_fmt_continue_fall()` 归一化布尔/是/已续/T/AC/AL/true/1 等多种标记为 `'是'/'否'/'-'`
- **消费端归一化**: `app.py` 与 `db.py` 统一用 `is_continue_fall()` 判断，避免未来因 Excel 公式(data_only)返回布尔值导致统计丢失
- **数据验证**: 最新校准 run 续报 PV 恢复为 86/1054，总续报率 8.2%；输出台帐 T/AC/AL 列 86 个非空值
- **前端变动概览重构**: `index.html` 新增 `formatStudentChanges()`/`abbrevChangeType()`，把原来的长 summary 字符串改为按学员卡片展示，每个学科一行：
  - 退费：学科 + 退费类型（如“双语素养：退费(课前退)”）
  - 调班：学科 + 变更字段简称（班/讲/室/时/期/型），如“雪球思维：调班(班/讲/室/时/期)”
  - 新增报科：学科 + “新增报科”
- **样式**: 新增 `.change-grid` / `.change-card` / `.ch-lines` 等 CSS，绿/红/橙三色卡片，信息更紧凑清晰

## 简化调班展示
- **用户逻辑**: 每个班的老师/教室/时段/期次是固定的，调班必然导致教室时段一起变，只需展示「期次变动」和「主讲变动」
- **修改**: `formatStudentChanges()` 中调班展示从 `调班(班/讲/室/时/期)` 改为精确匹配 `field==='period'` 和 `field==='teacher'`
- **格式**: `雪球思维：调班（零期→一期，主讲 陈中帅→何咏04）`；仅期次变则 `调班（零期→一期）`；仅主讲变则 `调班（主讲 张三→李四）`

## 李奕萱异常数据修正
- **问题**: 李奕萱(4815971458)三科联报，退了数学和语文。但数学退费订单在系统中误标为英语(双语素养)，导致出现两个英语订单(一退一在读)、缺少数学订单

- **前端**: 校准结果新增「⚠️ 异常数据修正」区块，橙色警告样式展示修正详情
- **补充**: 数学退费班级信息手动更正为 3132880（零期/启能班/陈中帅/教室2/15:50-18:00），`_fix_known_anomalies()` 中同时纠正 subject + class_id/period/teacher/room/time_slot/class_type
