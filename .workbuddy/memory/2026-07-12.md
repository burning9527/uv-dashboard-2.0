- **真正的根因（关键，推翻前两次尝试）**：Chart.js 的插件级 scale 钩子 `afterDataLimits(scale, args)` 实际收到的第一个参数是 **Chart 实例而非 scale**（实测 `context.constructor===Chart`，需通过 `context.scale` 取真实 scale）；且 `beforeUpdate` 里挂在 `chart.options.scales.y.suggestedMax` / `scale.afterDataLimits` 上的写法都 **不会** 被 Chart.js 调用（前者只是"建议"会被再次 nice 取整覆盖，后者 scale 选项对象上的钩子不触发）。所以前两次"还是遮挡"是 hook 根本没跑。
- **最终可靠方案**：在插件 `beforeUpdate(chart)` 中直接计算 bar 数据集峰值（堆叠按 index 累加），对未显式设 max 的 `y` 轴直接写 `chart.options.scales.y.max = _niceMax(dataMax * 1.15)`（显式 max Chart.js 必然采用，不会被覆盖），百分比轴 `y1` 已显式设 max=100 自动跳过。已在 Node + 真实 chart.js 实例化验证：峰值 305 → y.max=360（留白 15.3%），y1 保持 100。
