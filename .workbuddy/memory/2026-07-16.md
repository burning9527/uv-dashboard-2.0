## 多分校数据共存完整修复（深圳数据被识别为0 → 1217）
- **根因**：深圳学员的 subjects_json 是**旧格式**（key 无 `#suffix`，如 `雪球思维` 而非 `雪球思维#0`）：
  - `status=""`（空字符串）→ 引擎 `is_active = '在读' in status` 和 `is_refund = status and ... '在读' not in status` 两者均为 False → 整行被 continue
  - `period=""` → 矩阵行级 `plist=[]` → 整行被 continue（不计入教学点×期次）
  - 旧格式的 `_upgrade_old_subjects()` 函数虽然补全 `subject/season/grade/class_mode`，但**没补全空 `status` 和 `period`**
- **修复**（`repository/__init__.py` `_upgrade_old_subjects`）：
  - `if not line.get('status') or not line.get('status').strip(): line['status'] = '在读'`
  - `if not (line.get('period') or '').strip(): line['period'] = '一期'`（短学期兜底，矩阵需此字段）
- **手动触发 rebuild**：开发模式下 launchd 用项目目录 DB（`/Users/wangboning/WorkBuddy/2026-06-29-13-19-59/uv-dashboard-2.0/uv_dashboard.db`），与 `~/Library/Application Support/UV Dashboard 2.0/uv_dashboard.db` 是两个独立 DB。重建时必须 `repository.set_data_dir()` 指向正确路径，否则写错 DB。
- **端到端验证**：
  - 全局：active_uv=2316, total_uv=2453
  - 广州：active_uv=1099, total_uv=1236, refund_uv=137
  - 深圳：active_uv=1217, total_uv=1217, refund_uv=0（旧格式无退费信息，全部归为在读）
  - 深圳矩阵：points=['南山书城', '南山大新', '宝安灵芝'], total_uv=610, total_pv=2440
- **坑点**：
  - launchd `kickstart -k` 偶尔无法真正重启（PID 不变）；用 `launchctl kill SIGTERM` 更可靠
  - curl URL 编码：`深圳` 是 `%E6%B7%B1%E5%9C%B3`（圳=`%E5%9C%B3`），不是 `%E5%B7%B3`（那是「右」字）
