## 0. 决策摘要（用户确认的 8 项）

| # | 问题 | 决策 |
|---|---|---|
| Q1 | 渠道维度 | 仅用 **订单线索二级渠道(AM)**；商机渠道(AO)不进筛选器，一级渠道仅作 UI 分组 |
| Q2 | 课程类型 | **增加**「课程类型(特惠/系统)」+「产品类型(转化/正价)」筛选项 |
| Q3 | 入班类型 | **不做筛选器**；新生/老生/老生拓科目 作为**派生标签**放在课程详细，用于图表差异分析 |
| Q4 | 续报分母 | 使用 **BW 列（含联报）** 作分母；U2 规则(CG)暂不启用（启用时另行通知） |
| Q5 | 年级双轴 | **不单独成筛选器**；校内年级+公立校放进**学员信息卡片** |
| Q6 | 顾问聚合 | 按 **成单归属人 uname(列34)** 聚合，显示用 name(列35) |
| Q7 | 在读状态 | 仅 **在读 / 退费**；退费细分 **课前退 / 课后退**；丢弃「出班」口径（无结课场景） |
| Q8 | 学科命名 | 主名用**产品原名**（雪球思维/双语素养/悦读创作/雪球科学）；小标签用**学科简称**（数/语/英/物） |

## 2. 派生标签算法

### 2.1 学员类型（新生 / 老生续报 / 老生拓科目）— Q3
在课程详细/学员卡片中用 `retain_last` + 跨科目续报组合算出：
- **新生**：retain_last = 否（上季非老生）。
- **老生续报**：retain_last = 是 且 该科目续下一学季(renew_next=1)。
- **老生拓科目**：retain_last = 是 且 该学员在本学季扩报了其他科目（跨科目判定）。
> 仅用于标签展示与图表差异分析，**不做筛选器**。

### 2.2 期次显示名
period=无 时按 season 显示：秋季→秋，春季→春，暑假→暑，寒假→寒；其余期次原样。

### 2.3 学科双名
主名=产品原名（雪球思维…）；小标签/色块=学科简称（数/语/英/物）。

## 3. 对源字典的修正（已采纳）

1. **入班类型(列72)释义错**：实际=标内PV/尾单PV（报名时点），非「老生续报/纯新生/扩科目」。后者为 §2.1 派生标签。
2. **成单归属人(列35)释义错**：应为「姓名，未必唯一」；「唯一」属列34 uname。
3. **班级结束日期(列23)笔误**：「改时间」→「该时间」。
4. **续下一学季类型(列47)**：实际 5 类非空（含「系统课续系统课/特惠课续系统课」笔误），非「3类+历史」。归一为 系统课系统课/特惠课系统课/系统课特惠课。
5. **是否联报(列74)为空**且与「联报下下一学季(列48)」重叠，重复对疑为 48/74（非49）。
