三、功能闭环缺口

Functionality Gaps

🔧 按优先级排列 P0 高P1 中P2 低

  • P0
    数据导出全面缺失
    后端 /api/overview/export 与 /api/share/deploy 均为 TODO 占位；学员检索、期次矩阵、排班看板均无导出 Excel。对"数据工具"而言是硬伤，用户无法把看板结论带走。
  • P0
    校准无回滚 / 切换当前 run
    运行监控仅能"查看历史 run 详情"，无法将某次 run 设为当前数据基线。一次错误校准即污染全看板，且无法从界面恢复，风险高。
  • P1
    跨 run 对比缺失
    校准的核心价值是"这次相对上次变了什么"，但运行监控只列 run 列表，无 run-vs-run 增量可视化（新增 / 退费 / 续报率漂移），错失最有说服力的洞察。
  • P1
    云端分享未接通
    "生成分享"仅产出本地 HTML 可打开，/api/share/deploy 未实现，无法生成可外发链接，分享半径受限。
  • P2
    自动 / 定时校准
    当前必须手动上传触发；若招生产线每日出单，缺定时任务或自动拉取，运营成本高。

四、UI / UX 待打磨项

Interface & Interaction

High 高优先

  • !
    明暗风格割裂
    主应用为浅色品牌风，但分享页是 1.0 暗色主题，打开链接看到"另一个产品"，认知断档。建议分享页提供浅色品牌主题，或主应用支持暗色模式。
  • !
    错误态缺位
    接口失败多为静默 console.error，界面无 toast / 错误边界；用户可能面对空白卡片或旧数据而无感知。
  • !
    数据时效弱提示
    最新校准时间藏在 header 角落小字，无"数据 N 小时前更新"或"重新校准"入口，用户难判断看板是否新鲜。

Medium 中优先

  • ~
    主看板图表单薄
    一屏总览仅 2 张图（堆叠趋势 + 拉新月度），缺续报率仪表、学科/教学点构成环图、转化漏斗等"一眼洞察"组件，总览感不足。
  • ~
    筛选范式混合
    详情 Tab 用 chips 即时刷新，看板用多选下拉 + "查询"按钮，两种范式并存易增加认知负担；窄屏下 8 项筛选栏换行堆叠无折叠。
  • ~
    配色局部不一致
    主品牌蓝紫，但"生成分享/分享卡片"按钮用绿 #1D9E75；图例颜色硬编码在 HTML，与 config 色板不同步，改主题易漏。
  • ~
    空态简陋
    .no-data 仅一行灰字；无筛选结果 / 无学员时应给引导插画或操作建议。

五、信息架构与数据可视化

IA & Data Viz
  • A
    Tab 语义可优化
    4 主Tab + 5 子Tab 信息量大；"数据明细"把查看型（矩阵/排班）与下钻型（主讲/顾问/学员）混排，建议分组或加图标语义区分"看板"与"检索"。
  • B
    图表类型单一
    目前仅"堆叠柱 + 折线"。建议补充：续报率仪表盘、学科/教学点构成环图、转化漏斗、期次 × 指标热力图，丰富叙事层次。
  • C
    口径标注需显式
    主趋势图用 PV、KPI 用 UV/PV 混用，应在界面明确标注每一处口径，避免业务方质疑"数字对不上"。
  • D
    分享页与看板口径对齐
    分享页月度图忽略期次筛选（periods=''），与看板筛选不完全联动，需明示或对齐。
